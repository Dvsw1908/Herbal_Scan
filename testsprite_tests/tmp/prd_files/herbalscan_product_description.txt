====================================================
HERBALSCAN - PRODUCT DESCRIPTION FOR TESTING
====================================================

PROJECT NAME: HerbalScan
TYPE: Mobile Application (Flutter) + REST API Backend (FastAPI + PyTorch)
PURPOSE: Herbal leaf classification system for academic research (undergraduate thesis)

----------------------------------------------------
1. PURPOSE & OVERVIEW
----------------------------------------------------
HerbalScan is a mobile application that classifies herbal plant leaves using a
ShuffleNetV2 deep learning model. The app allows users to take a photo or upload
an image of a herbal leaf, sends it to a FastAPI backend for inference, and
displays the predicted plant species along with a confidence score. Results are
stored locally in SQLite and also recorded in the backend database.

The system supports 16 classes (8 herbal plant species x 2 conditions: healthy
and damaged/rusak), based on two experimental scenarios:
  - Scenario 1: Training with background vs. without background (remove-BG)
  - Scenario 2: Healthy leaf vs. Damaged leaf (identified by "rusak" in folder name)

----------------------------------------------------
2. BACKEND API - FEATURES & EXPECTED BEHAVIOR
----------------------------------------------------
Base URL: http://localhost:8000
Framework: FastAPI (Python)
Database: SQLite (predictions.db)
Model: ShuffleNetV2 (.pth file loaded via inference.py)

ENDPOINTS:

  [GET] /
    - Purpose: Health check
    - Expected response: { "status": "HerbalScan backend siap", "version": "1.0.0" }
    - Expected status: 200 OK

  [POST] /predict
    - Purpose: Upload a leaf image and get classification result
    - Input: multipart/form-data with field "file" (image: JPEG / PNG / WEBP)
    - Max file size: 10 MB
    - Expected response:
        {
          "id": <integer>,
          "predicted_class": <string>,
          "confidence_score": <float, 4 decimal places>
        }
    - Error cases:
        - Unsupported file type (not jpeg/png/webp) → HTTP 415
        - File size > 10 MB → HTTP 413
        - Corrupt/invalid image bytes → HTTP 400
        - No file provided → HTTP 422 (validation error)

  [GET] /history
    - Purpose: Retrieve all prediction records from the database
    - Expected response: list of prediction objects
    - Each record contains: id, image_path, predicted_class, confidence_score, timestamp
    - Expected status: 200 OK

  [DELETE] /history/{prediction_id}
    - Purpose: Delete a specific prediction record by ID
    - Expected response (success): { "message": "Data berhasil dihapus." }
    - Expected response (not found): HTTP 404 with detail "Data tidak ditemukan."

CORS: All origins allowed (allow_origins=["*"])

----------------------------------------------------
3. FLUTTER MOBILE APP - FEATURES & EXPECTED BEHAVIOR
----------------------------------------------------
Framework: Flutter (Dart)
State Management: Provider (PredictionProvider)
Local Storage: SQLite via sqflite
Navigation: Material 3 NavigationBar (3 tabs)

PAGES & FEATURES:

  [Home Page]
    - User can pick an image from camera or gallery (image_picker)
    - Selected image is sent to backend POST /predict
    - Prediction result (class + confidence) is shown in a dialog or result view
    - Successful prediction is saved to local SQLite database

  [History Page]
    - Displays list of all past predictions from local SQLite
    - Each item shows: image thumbnail, predicted class, confidence score, timestamp
    - User can delete a single prediction item
    - User can clear all history at once

  [Settings Page]
    - Tutorial dialog: explains how to use the app
    - Dataset Info dialog: explains the 16 herbal leaf classes and scenarios
    - About dialog: project and developer information
    - Model Info dialog: ShuffleNetV2 architecture and training details

  [Plant Detail Page]
    - Shows detailed information about a specific herbal plant species
    - Accessible from Home or History context

----------------------------------------------------
4. DATA MODEL
----------------------------------------------------
Prediction record:
  - id: integer (primary key, auto-increment)
  - image_path: string (filename or local file path)
  - predicted_class: string (one of 16 herbal leaf classes)
  - confidence_score: float (0.0 to 1.0)
  - timestamp: datetime string

6. BACKEND TESTING PRIORITY CHECKLIST
----------------------------------------------------
  1.  GET /  → returns 200 with correct status message
  2.  POST /predict with valid JPEG → returns id, predicted_class, confidence_score
  3.  POST /predict with valid PNG → returns correct prediction
  4.  POST /predict with invalid type (text/plain) → returns 415
  5.  POST /predict with file > 10MB → returns 413
  6.  POST /predict with corrupt image bytes → returns 400
  7.  POST /predict with no file → returns 422
  8.  POST /predict response contains correct data types (id=int, class=str, score=float)
  9.  GET /history → returns array (empty or populated)
  10. GET /history → newly saved prediction appears after POST /predict
  11. DELETE /history/{valid_id} → returns success message, 200
  12. DELETE /history/{invalid_id} → returns 404
  13. CORS headers present in all responses
